# Mac Fleet Usage

This folder keeps the local Mac setup for using `ccusage` across multiple Macs.
It is intentionally separate from the product code.

## Model

Each Mac exports its local Codex logs into a shared fleet folder:

```text
$CCUSAGE_FLEET_ROOT/
  codex/
    mac-mini-m4-1/
      sessions/
      archived_sessions/
      config.toml
    macbook-air-m4/
      sessions/
      archived_sessions/
      config.toml
```

`ccusage` already accepts comma-separated `CODEX_HOME` directories, so combined
reports are generated by pointing it at every per-Mac folder. Individual reports
are generated by pointing it at one per-Mac folder.

Use a sync backend you control for `CCUSAGE_FLEET_ROOT`, such as iCloud Drive,
Syncthing, a NAS folder, or an SSH/rsync destination. Do not put exported Codex
logs in git.

## Install On Each Mac

Clone your fork and install the CLI:

```sh
git clone git@github.com:nibrwr/ccusage.git ~/Development/ccusage
cd ~/Development/ccusage
ops/macos/install-ccusage.zsh
```

Pick the machine config for that Mac:

```sh
source ops/macos/machines/mac-mini-m4-1.env
# or:
source ops/macos/machines/macbook-air-m4.env
```

Export that Mac's Codex logs:

```sh
ops/macos/export-codex-logs.zsh
```

## Reports

Combined usage across every exported Mac:

```sh
ops/macos/report-codex-fleet.zsh monthly
ops/macos/report-codex-fleet.zsh daily
ops/macos/report-codex-fleet.zsh session
```

Individual Mac usage:

```sh
ops/macos/report-codex-fleet.zsh --machine mac-mini-m4-1 monthly
ops/macos/report-codex-fleet.zsh --machine macbook-air-m4 monthly
```

Pass normal `ccusage codex` options after the report name:

```sh
ops/macos/report-codex-fleet.zsh monthly --json
ops/macos/report-codex-fleet.zsh --machine mac-mini-m4-1 daily --since 2026-05-01
```

## Scheduled Export

After sourcing the correct machine config, install a launchd job:

```sh
ops/macos/install-export-launchd.zsh
```

The job runs every 15 minutes by default. Override the interval with
`CCUSAGE_EXPORT_INTERVAL_SECONDS`.
