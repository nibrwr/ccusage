# Mac Fleet Usage

This folder keeps the local Mac setup for using `ccusage` across multiple Macs.
It is intentionally separate from the product code.

## Model

Each Mac exports its local Codex logs into a shared fleet folder:

```text
$CCUSAGE_FLEET_ROOT/
  codex/
    mac-mini-m4-1/
      sessions/
      archived_sessions/
      config.toml
    macbook-air-m4/
      sessions/
      archived_sessions/
      config.toml
```

`ccusage` already accepts comma-separated `CODEX_HOME` directories, so combined
reports are generated by pointing it at every per-Mac folder. Individual reports
are generated by pointing it at one per-Mac folder.

By default, the scripts export to this local folder:

```sh
$HOME/Library/Application Support/ccusage-fleet
```

That is safe for a single Mac but does not sync across machines. To combine
multiple Macs, explicitly choose a shared `CCUSAGE_FLEET_ROOT`, such as iCloud
Drive, Syncthing, a NAS folder, or an SSH/rsync destination. Do not put exported
Codex logs in git.

Codex logs can contain sensitive prompt, path, and code context. Treat the fleet
folder as private data.

## Install On Each Mac

Clone your fork and install the CLI:

```sh
git clone git@github.com:nibrwr/ccusage.git ~/Development/ccusage
cd ~/Development/ccusage
ops/macos/install-ccusage.zsh
```

If the install script reports `cargo was not found`, install Rust first:

```sh
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
exec zsh
ops/macos/install-ccusage.zsh
```

Pick the machine config for that Mac:

```sh
source ops/macos/machines/mac-mini-m4-1.env
# or:
source ops/macos/machines/macbook-air-m4.env
```

Machine configs also enable local refresh before report commands. Pass
`--no-refresh` to a report command when you only want to read the already
exported fleet data.

Optionally opt into iCloud Drive as the shared fleet folder:

```sh
source ops/macos/fleet-roots/icloud.env
```

Export that Mac's Codex logs:

```sh
ops/macos/export-codex-logs.zsh
```

If you previously exported to the local default
`~/Library/Application Support/ccusage-fleet`, source your shared fleet root
profile and rerun the export so the logs land in the shared folder.

## Reports

Combined usage across every exported Mac:

```sh
ops/macos/report-codex-fleet.zsh monthly
ops/macos/report-codex-fleet.zsh daily
ops/macos/report-codex-fleet.zsh session
```

Usage broken out by Mac:

```sh
ops/macos/report-codex-fleet-by-machine.zsh monthly
ops/macos/report-codex-fleet-by-machine.zsh weekly
ops/macos/report-codex-fleet-by-machine.zsh daily --since 2026-05-01
```

Individual Mac usage:

```sh
ops/macos/report-codex-fleet.zsh --machine mac-mini-m4-1 monthly
ops/macos/report-codex-fleet.zsh --machine macbook-air-m4 monthly
```

Pass normal `ccusage codex` options after the report name:

```sh
ops/macos/report-codex-fleet.zsh monthly --json
ops/macos/report-codex-fleet.zsh --machine mac-mini-m4-1 daily --since 2026-05-01
```

## Scheduled Export

After sourcing the correct machine config, install a launchd job:

```sh
ops/macos/install-export-launchd.zsh
```

The job runs every 15 minutes by default. Override the interval with
`CCUSAGE_EXPORT_INTERVAL_SECONDS`.

For fresher shared reports, use a shorter interval when installing the job:

```sh
CCUSAGE_EXPORT_INTERVAL_SECONDS=60 ops/macos/install-export-launchd.zsh
```

Check whether the job is loaded and when each Mac last exported data:

```sh
ops/macos/status-codex-fleet.zsh
```

## Scheduled Reports

Install weekly and monthly by-machine report delivery from the reporting Mac's
GUI session:

```sh
source ops/macos/fleet-roots/icloud.env
ops/macos/install-report-launchd.zsh
```

Reports are saved under `$CCUSAGE_FLEET_ROOT/reports` and a macOS notification
is posted when each report is generated. By default, weekly reports run Monday
at 08:00 and monthly reports run on day 1 at 08:00.

Generate a report immediately:

```sh
ops/macos/generate-codex-fleet-report.zsh weekly
ops/macos/generate-codex-fleet-report.zsh monthly
```
